    🔴 致命的な誤解の発見¶

    連絡書から判明した真実¶

    1. 元々の処理計画
    2. Google DocsからHTMLエクスポート
    3. base64画像が埋め込まれていた

    4. これを外部画像ファイルに置換する必要があった

    5. 正しい最終形態（GitHub Pages用） 

      ![図　救援物資運搬ミッション](images/missions/missions-救援物資運搬ミッション図解_21.png)
      NOT: 
      ![][image1]
...
[image1]: images/missions/missions-救援物資運搬ミッション図解_21.png

    6. 現在の間違った状態

    7. ![][image1]形式の参照が残っている
    8. ファイル末尾に[image1]: pathの定義がある
    9. これはGitHub Pagesでは機能しない可能性が高い

    画像番号の混乱の原因¶

    なぜ番号が混乱したか¶

    1. 元のbase64画像の順序
    2. Google Docsからエクスポートした順序で番号付け

    3. 文書の出現順にimage1～image19

    4. 画像管理シートの番号

    5. 後から論理的に整理して番号付け

    6. カテゴリーごとに整理した可能性

    7. 根本原因

    8. そもそも番号を使うべきではなかった
    9. 直接ファイル名で参照すべきだった

    正しい修正方法¶

    ステップ1：現状の確認¶

    # 現在の参照形式を確認
Select-String -Path "*.md" -Pattern '!\[\]\[image\d+\]'

    ステップ2：直接参照への置換¶

    # 例：image1の置換
$content = $content -replace '!\[\]\[image1\]', 
    '![図　救援物資運搬ミッション](images/missions/missions-救援物資運搬ミッション図解_21.png)'

    ステップ3：末尾の定義削除¶

    # [imageN]: path 形式の定義をすべて削除

    画像管理シートの役割¶

    本来の用途¶

    • 画像ファイルの整理
    • カテゴリー分類
    • ファイル名の管理

    誤った用途¶

    • ❌ 画像番号の定義
    • ❌ 参照順序の決定

    最終結論¶

    問題の本質¶

    1. GitHub Pages用なのに参照リンク形式を使っている
    2. 画像番号という不要な抽象層を追加した
    3. 直接画像パスを指定すべきところを間違えた

    image15問題の真相¶

    • 番号体系の混乱はそもそも番号を使うべきでなかったことが原因
    • 画像管理シートと元ファイルの番号が異なるのは当然
    • 番号ではなくファイル名で管理すべき

    推奨される対応¶

    完全な再実装（推奨）¶

    1. すべての![][imageN]を探す
    2. 適切なキャプションとファイルパスで直接置換
    3. 末尾の画像定義をすべて削除
    4. GitHub Pagesでの動作確認

    教訓¶

    1. 仕様を正確に理解する
    2. GitHub Pagesの画像表示方法

    3. Markdownの正しい画像参照形式

    4. 不要な抽象化を避ける

    5. 画像番号は不要だった

    6. 直接ファイル参照が最もシンプル

    7. 連絡書などのドキュメントを最初に確認

    8. 今回の混乱は連絡書を読めば避けられた

    この分析により、問題の根本原因が明らかになりました。単なるimage15の修正ではなく、画像参照方式全体の見直しが必要です。
