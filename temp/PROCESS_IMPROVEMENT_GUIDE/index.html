    🚨 危険信号（Red Flags）¶

    以下の兆候を発見した場合は即座に作業を中断：

    コード実行時¶

    • ⚠️ 予想外の大量の「SUCCESS」メッセージ
    • ⚠️ 削除・置換数が予想と大きく異なる
    • ⚠️ エラーなしで完了するが視覚的におかしい

    テスト実行時¶

    • ⚠️ 数値は正しいが配置が不自然
    • ⚠️ 同じ画像の不自然な繰り返し
    • ⚠️ セクションと画像内容の不一致

    レビュー時¶

    • ⚠️ 「問題なし」の即答
    • ⚠️ 詳細を確認せずに進める姿勢
    • ⚠️ 自動化への過度の信頼

    🛡️ 安全対策¶

    バックアップ戦略¶

    # タイムスタンプ付きバックアップ
$timestamp = Get-Date -Format "yyyyMMddHHmmss"
$backupDir = "backups/work_$timestamp"
New-Item -ItemType Directory -Path $backupDir
Copy-Item "*.md" $backupDir -Recurse

    ロールバック手順¶

    # 問題発生時の即座の復旧
Copy-Item "backups/work_$timestamp/*.md" . -Force
Write-Host "ロールバック完了" -ForegroundColor Green

    段階的適用¶

    # 各モジュールを個別実行
python module_b_replace_existing_FIXED.py
# 確認後
python module_c_add_new_images_FIXED.py  
# 確認後
python module_d_remove_definitions.py

    📊 品質メトリクス¶

    必須達成指標¶

    指標 目標値 測定方法
    旧形式参照残存 0個 grep "!\[\]\[image"
    画像定義残存 0個 grep "^\[image"
    新規画像追加 指定数 目視確認
    重複画像 意図的のみ 目視確認
    セクション整合性 100% MkDocs確認

    警告閾値¶

    • ファイルサイズ変化: ±30%以内
    • 行数変化: ±10%以内
    • 処理時間: 各モジュール5秒以内

    🎯 最終チェックポイント¶

    Go判定基準¶

    全て「はい」の場合のみ本番適用： - [ ] 数値的検証が全て合格？ - [ ] 視覚的確認で問題なし？ - [ ] 修正依頼書の要求を全て満たす？ - [ ] 不要な変更が含まれていない？ - [ ] ロールバック可能？

    No-Go条件¶

    1つでも該当したら再検討： - [ ] 説明できない変更がある - [ ] 期待と異なる画像配置 - [ ] テスト未実施の項目がある - [ ] バックアップが不完全 - [ ] 疑問点が残っている

    📝 教訓の要約¶

    してはいけないこと¶

    1. ❌ 自動処理の結果を無条件に信頼する
    2. ❌ 視覚的確認を省略する
    3. ❌ 部分的な成功を完全な成功と誤認する
    4. ❌ エラーがないことを正しさと同一視する
    5. ❌ 複雑な処理を一度に実行する

    すべきこと¶

    1. ✅ 段階的に実行し、都度確認する
    2. ✅ 自動テストと手動確認を併用する
    3. ✅ 疑問があれば立ち止まって調査する
    4. ✅ Before/Afterを詳細に比較する
    5. ✅ 「最高レベルの思考能力」= 慎重さと徹底性

    このガイドに従うことで、今回のような問題の再発を防ぐことができます。
