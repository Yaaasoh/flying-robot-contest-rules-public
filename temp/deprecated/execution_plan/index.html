    飛行ロボコン21 画像表示問題調査 実行計画書¶

    作成日時: 2025/01/11 実行者: Claude Code

    1. 調査目的¶

    21FlyRobo文書の画像表示問題を徹底調査し、問題の全容を把握して修正方針を確立する

    2. タスク実行順序と成果物¶

    タスク1: 修正履歴の完全調査（推定時間: 15分）¶

    優先度: 最優先 成果物: - modification_history.txt - 修正前後の画像タグ差分リスト - image_tag_changes.csv - 画像番号ごとの変更履歴表

    タスク2: キャプション調査（推定時間: 10分）¶

    優先度: 最優先 成果物: - caption_analysis.txt - キャプションの新旧対比表 - captions_to_remove.txt - 削除すべきキャプションリスト - captions_to_keep.txt - 保持すべきキャプションリスト

    タスク3: 画像管理シートとの整合性確認（推定時間: 15分）¶

    優先度: 高 成果物: - image_mapping_validation.csv - 現在の画像定義と正しいマッピングの比較 - mapping_errors.txt - 不整合箇所のリスト - correct_image_definitions.txt - 修正すべき画像定義

    タスク4: 部分修正による全体破壊の原因分析（推定時間: 20分）¶

    優先度: 中 成果物: - root_cause_analysis.txt - 原因分析レポート - modification_timeline.csv - 修正作業のタイムライン - impact_analysis.txt - 影響範囲の詳細分析

    タスク5: 修正前後の詳細比較（推定時間: 20分）¶

    優先度: 中 成果物: - image_tag_locations.csv - 各画像タグの使用箇所一覧 - image_definitions_comparison.txt - 画像定義部分の詳細比較 - section_impact_report.txt - セクション別の影響度レポート

    3. 総作業時間見積もり¶

    • 合計: 約80分（1時間20分）
    • バッファ時間: 20分
    • 予想完了時間: 約2時間

    4. 実行上の注意事項¶

    1. ファイルサイズが大きいため（約967KB）、部分的に処理
    2. すべてのファイルはUTF-8エンコーディングで処理
    3. 画像番号の重複（image15とimage16）に特別な注意
    4. 修正作業自体は実施せず、調査・分析のみ実施
    5. すべての成果物はdocs/temp/に保存

    5. 重要な確認ポイント¶

    • image6とimage15の重複参照問題
    • 「図　」キャプションの追加箇所
    • 画像定義部分の整合性
    • セクション間での画像配置の適切性

    6. 次のステップ（調査完了後）¶

    1. 調査結果の総合レポート作成
    2. 修正方針の策定
    3. 段階的修正計画の立案
    4. 再発防止策の文書化
