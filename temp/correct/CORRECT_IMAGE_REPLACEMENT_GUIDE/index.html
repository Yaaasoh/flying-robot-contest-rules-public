    1. 正しい理解¶

    GitHub Pages用の画像参照形式¶

    正しい形式： 

    ![図　キャプション](images/category/filename.png)

    間違った形式（使用禁止）： 

    ![][image1]
...
[image1]: images/category/filename.png

    なぜ直接参照が正しいのか¶

    1. GitHub Pagesで確実に動作
    2. 管理がシンプル
    3. 画像とキャプションの関係が明確
    4. 不要な番号管理が不要

    2. 現状の問題¶

    現在のファイルの状態¶

    • ![][imageN] 形式の参照が19箇所存在
    • ファイル末尾に [imageN]: path の定義が存在
    • これらをすべて直接参照形式に変更する必要がある

    4. 置換作業の手順¶

    Step 1: バックアップ作成¶

    Copy-Item "21FlyRobo_GeneralAutoUniqueMulti_Regulations_text.md" `
          "21FlyRobo_GeneralAutoUniqueMulti_Regulations_text.md.backup_$(Get-Date -Format 'yyyyMMdd_HHmmss')"

    Step 2: 各画像参照を置換¶

    # PowerShellスクリプト例
$content = Get-Content "21FlyRobo_GeneralAutoUniqueMulti_Regulations_text.md" -Raw -Encoding UTF8

# 各画像を正しい形式に置換
$replacements = @{
    '![][image1]' = '![図　一般部門・自動操縦部門・ユニークデザイン部門のフィールド](images/field/field-一般部門_自動操縦部門_ユニークデザイン部門競技エリア_21.png)'
    # ... 他の置換も同様
}

foreach ($old in $replacements.Keys) {
    $content = $content -replace [regex]::Escape($old), $replacements[$old]
}

    Step 3: 末尾の画像定義を削除¶

    ファイル末尾の以下の形式の行をすべて削除： 

    [image1]: images/missions/missions-救援物資運搬ミッション図解_21.png
[image2]: images/missions/missions-マルチコプター部門８の字飛行ミッション図解_21.png
...
[image19]: images/missions/missions-自動離着陸ミッション図解_21.png

    Step 4: 検証¶

    1. すべての ![][imageN] 形式が置換されたか確認
    2. 画像定義がすべて削除されたか確認
    3. MkDocsでプレビューして画像が表示されるか確認

    5. 検証チェックリスト¶

    □ バックアップを作成した
□ すべての ![][imageN] 形式を置換した
□ 末尾の画像定義をすべて削除した
□ 画像ファイルが正しいパスに存在する
□ MkDocsでプレビューして画像が表示される
□ GitHub Pagesにデプロイして動作確認

    6. トラブルシューティング¶

    画像が表示されない場合¶

    1. パスが正しいか確認（大文字小文字も含めて）
    2. 画像ファイルが存在するか確認
    3. 相対パスが正しいか確認

    文字化けする場合¶

    1. ファイルのエンコーディングがUTF-8か確認
    2. 日本語ファイル名が正しく扱われているか確認

    7. 最終確認¶

    成功の基準¶

    • すべての画像が正しく表示される
    • 参照リンク形式の記述が残っていない
    • GitHub Pagesで正常に動作する

    重要: この方法が唯一の正しいアプローチです。画像番号を使った管理は行いません。
