    第21回飛行ロボコン画像管理 - base64除去処理に関する連絡書¶

    作成者: Claude
    宛先: 飛行ロボコンプロジェクト関係者各位
    件名: 第21回飛行ロボコン画像管理におけるbase64除去処理の実施について

    1. 経緯と背景¶

    1.1 プロジェクトの経過¶

    • 2025年8月5日～8月7日: Phase 1作業としてGoogle DocsからHTMLエクスポート経由で画像を抽出
    • 一般部門19枚、ビギナー部門8枚の画像ファイルを取得
    • base64形式で埋め込まれた状態で作業が中断
    • 2025年9月8日: 作業再開、PowerShellによる画像参照数の正確な検証を実施
    • 2025年1月29日: 画像ファイルの配置完了（合計22枚）、ディレクトリ構造の整備完了

    1.2 発生した問題¶

    MDファイル内に埋め込まれた大量のbase64画像データにより、以下の深刻な問題が発生しました： - ファイルサイズの肥大化: 一般部門967KB、ビギナー部門352KB - 処理困難性: Claudeでの処理時に生成上限に到達するリスク - 編集効率の低下: テキストエディタでの編集が困難 - Git管理の非効率化: 巨大なテキストファイルによるバージョン管理の問題

    2. base64除去処理の実施¶

    2.1 処理の概要¶

    ファイルサイズを劇的に削減し、編集・管理を容易にするため、base64画像データを除去する処理を実施しました。

    2.2 使用したPowerShellスクリプト¶

    # base64画像を除去した編集用ファイルを作成する関数
function Create-CleanEditFile {
    param(
        [string]$SourceFile,
        [string]$OutputFile
    )

    # ファイル内容を読み込み
    $content = Get-Content $SourceFile -Encoding UTF8

    # base64画像定義の開始を検出
    $cleanContent = @()
    $inBase64Section = $false

    foreach ($line in $content) {
        # base64画像定義の開始を検出
        if ($line -match '^\[image\d+\]: <data:image') {
            $inBase64Section = $true
            # 画像参照プレースホルダーを挿入
            $imageNum = [regex]::Match($line, '\[image(\d+)\]').Groups[1].Value
            $cleanContent += "[image$imageNum]: <!-- TO BE REPLACED WITH ACTUAL PATH -->"
        }
        elseif (-not $inBase64Section) {
            $cleanContent += $line
        }
    }

    # クリーンファイルを保存
    $cleanContent | Out-File $OutputFile -Encoding UTF8

    # サイズ比較
    $originalSize = (Get-Item $SourceFile).Length / 1KB
    $cleanSize = (Get-Item $OutputFile).Length / 1KB

    Write-Host "Original: $([Math]::Round($originalSize, 2)) KB" -ForegroundColor Red
    Write-Host "Clean: $([Math]::Round($cleanSize, 2)) KB" -ForegroundColor Green
    Write-Host "Reduction: $([Math]::Round((($originalSize - $cleanSize) / $originalSize * 100), 2))%" -ForegroundColor Yellow
}

    2.3 処理結果¶

    • 一般部門: 944.69KB → 80.45KB（91.48%削減）
    • ビギナー部門: 344.24KB → 27.65KB（91.97%削減）

    5. 技術的な注意事項¶

    5.1 エンコーディング¶

    • すべての処理でUTF-8エンコーディングを使用
    • PowerShellでは -Encoding UTF8 パラメータを必須指定

    5.2 バックアップ戦略¶

    各処理段階でバックアップファイルを作成： - .backup_yyyyMMddHHmmss - タイムスタンプ付きバックアップ - .backup_base64 - base64含有の元ファイル - .backup_before_edit - 編集前の状態

    5.3 検証手順¶

    1. ファイルサイズの確認（90%以上の削減を確認）
    2. 画像参照の整合性確認（すべての[imageN]が適切に変換されているか）
    3. MkDocsでのプレビュー確認（画像が正しく表示されるか）
