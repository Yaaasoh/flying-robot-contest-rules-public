    1. 問題発生の経緯¶

    1.1 本日の修正作業の概要¶

    • 作業内容: 21FlyRobo文書の画像タグを管理シートに基づいて一括置換
    • 使用した方法: 画像番号（[image1]～[image19]）形式への統一
    • 具体的な変更内容:
    • 画像参照形式の変更（例：直接URL → [imageN]形式）
    • 画像定義部分（ファイル末尾）の書き換え
    • 「図　」キャプションの追加（これが問題の原因）
    • 作業時の状況:
    • 初期段階では正常に動作していた部分もあった
    • 部分修正を重ねるうちに全体的な不整合が発生
    • 画像番号の重複（image6とimage15が同じファイルを参照）が発生

    1.2 発生した問題¶

    1. 不要なキャプション追加
    2. 私が「図　」というキャプションを勝手に追加した箇所がある

    3. 元からあったキャプションとの区別が必要

    4. 画像参照の混乱

    5. 画像番号と実際の画像ファイルの対応が崩れた

    6. 同じ画像への重複参照（例: [image6]と[image15]が同じチキンラーメンmini）

    7. 画像挿入位置の誤り

    8. セクション間で画像が入れ替わっている
    9. 本来の位置とは異なる場所に画像が挿入されている

    1.3 問題の深刻さ¶

    • 部分的な修正が全体の崩壊を招いた
    • 画像管理シートとの整合性が失われた可能性
    • 元の正常な状態への復旧が困難な状況

    7. 反省点と教訓¶

    今回の失敗から学ぶべきこと¶

    1. 部分修正のリスク: 画像参照システムは相互依存性が高く、部分的な変更が全体に波及する
    2. 事前検証の不足: 画像番号の重複や整合性チェックが不十分だった
    3. キャプション追加の判断ミス: 元のフォーマットを十分に理解せずに「図　」キャプションを追加した
    4. 他ファイル参照の誤り: 20FlyRoboファイルを参考にするなど、関係ない情報を参照した

    今後の改善点¶

    1. 修正前の完全な状態把握（21FlyRoboファイル内での一貫性確認）
    2. 変更影響範囲の事前評価
    3. 段階的な修正と都度の動作確認
    4. オリジナルフォーマットの尊重
    5. 不必要なキャプション追加の回避
